# AJAX

是一种无需重新加载整个网页的情况下，能够更新部分网页的技术，通过后台与服务器交换数据，并能够更新部分网页内容

## XMLHttpRequest

创建一个XMLHttpRequest对象的实例

```
var xhr = new XMLHttpRequest();
xhr.open('GET','https://api.example.com/data',true);
xhr.setRequestHeader('Content-Type','application/json');

xhr.onload = function(){
	if(xhr.status >= 200 && xhr.status < 300){
		console.log('Response:', xhr.responseText);
	}else{
		console.error('Error:',xhr.statusText);
	}
};

xhr.onerror = function (){
	console.error('Request failed');
}

xhr.send();

```

## XHR请求

open规定请求的类型，url,是否异步处理

send f发送请求到服务器

```
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<script>
function loadXMLDoc()
{
	var xmlhttp;
	if (window.XMLHttpRequest)
	{
		// IE7+, Firefox, Chrome, Opera, Safari 浏览器执行代码
		xmlhttp=new XMLHttpRequest();
	}
	else
	{
		// IE6, IE5 浏览器执行代码
		xmlhttp=new ActiveXObject("Microsoft.XMLHTTP");
	}
	xmlhttp.onreadystatechange=function()
	{
		if (xmlhttp.readyState==4 && xmlhttp.status==200)
		{
			document.getElementById("myDiv").innerHTML=xmlhttp.responseText;
		}
	}
	xmlhttp.open("GET","/try/ajax/demo_get.php",true);
	xmlhttp.send();
}
</script>
</head>
<body>

<h2>AJAX</h2>
<button type="button" onclick="loadXMLDoc()">请求数据</button>
<div id="myDiv"></div>

</body>
</html>
```

/try/ajax/demo_get.php在浏览器中是一种合法的绝对路径，浏览器会自动补全协议，主机名和端口号

假设现在访问的页面是 `https://www.example.com/index.html`，那么浏览器会自动将这个请求补全为：
`https://www.example.com/try/ajax/demo_get.php`

可以异步请求其他网站的url，但通常拿不到数据，因为浏览器的同源策略，

如果页面是 `www.a.com`，想请求 `www.b.com` 的数据，浏览器虽然会把请求发出去（服务器甚至会收到），但浏览器在接收返回数据时，会检查响应头中的 `Access-Control-Allow-Origin` 字段。

如果 `www.b.com` 的响应头里没有明确允许 `www.a.com` 访问，浏览器就会报跨域错误（CORS 错误），`readyState` 可能走到 4，但 `status` 为 0，或者报错，拿不到 responseText。

**例外情况**：如果目标网站**开启了 CORS（跨域资源共享）**，或者使用 JSONP 方式（仅限 GET），则可以正常请求其他网站的 URL。

## webpack

现代js应用程序的静态模块打包器(module bundler)

### entry

入口会指示webpack应该使用哪个模块

### loader

让webpack处理那些非js文件

### plugins

打包优化，压缩，定义环境变量

```
const path = require('path');
 
module.exports = {
  mode: "development", // "production" | "development"
  // 选择 development 为开发模式， production 为生产模式
  entry: "./src/main.js",
  output: {
    filename: "bundle.js",
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        loader: "babel-loader",
        options: [
          presets: ["env"]
        ]
      }
    ]
  },
  plugins: [
    ...
  ]
}
```

上述例子构建了一个最简单的配置，webpack 会从入口 main.js 文件进行构建，通过 loader 进行js转换，输出一个为 bundle.js 的文件，至此一整个过程就构建完成。
